# 智学 AI 学习平台需求文档

> [!abstract] 文档概述
> 本文档记录智学平台的功能需求、优先级和实现状态。
> 作为个人项目，需求会持续更新和调整。
>
> **相关文档**：[[智学产品方案]] | [[技术架构方案]]

---

## 需求管理说明

> [!info] 优先级定义
> - **P0**：核心功能，MVP 必须实现
> - **P1**：重要功能，第一版完成后尽快实现
> - **P2**：增强功能，有时间再做
> - **P3**：未来规划，暂不考虑

> [!note] 状态定义
> - **待开发**：需求已确认，等待开发
> - **开发中**：正在实现
> - **已完成**：已上线可用
> - **已搁置**：暂时不做
> - **已取消**：不再需要

---

## 1. 核心功能需求（P0 - MVP）

> [!warning] MVP 原则
> 只实现最核心的学习闭环，快速验证产品价值。
> 复杂功能、可视化、统计分析等延后到 P1。

### 1.1 学习目标问答

**需求描述**
通过 AI 对话收集用户学习需求，生成结构化学习画像。

**功能点**
- [ ] AI 结构化提问（必答字段：主题、目标、水平、时间）
- [ ] 实时验证必填项
- [ ] 生成学习画像（JSON 格式）
- [ ] 保存学习画像到数据库

**验收标准**
- 必答字段未填完不能进入下一步
- 生成的画像包含所有必需信息
- 画像可以被后续模块正确读取

**状态**：待开发

---

### 1.2 资料上传与解析（简化版）

**需求描述**
用户上传学习资料，系统自动提取文本内容。

> [!tip] MVP 简化
> 第一版只支持 TXT 和 MD 文件，PDF 解析延后到 P1

**功能点**
- [ ] 支持上传 TXT/MD 文件
- [ ] 文件大小限制（10MB）
- [ ] 文件类型验证（白名单）
- [ ] 提取文本内容
- [ ] 解析失败提示

**验收标准**
- TXT 和 MD 文件能完整读取
- 解析失败时给出明确提示

**状态**：待开发

---

### 1.3 学习大纲生成（简化版）

**需求描述**
根据上传的资料和学习画像，生成结构化学习大纲。

> [!tip] MVP 简化
> 不做版本管理，只保留基础编辑功能

**功能点**
- [ ] 基于资料内容生成章节大纲
- [ ] 根据学习画像调整难度和深度
- [ ] 显示每章预计学习时长
- [ ] 支持手动编辑大纲（增删改章节）
- [ ] 支持调整章节顺序

**验收标准**
- 生成的大纲逻辑清晰，有层次
- 章节数量合理（5-10 章）
- 手动编辑后能正确保存

**状态**：待开发

---

### 1.4 课程内容生成（简化版）

**需求描述**
根据大纲和资料，生成每个章节的学习内容。

> [!tip] MVP 简化
> 使用固定模板生成，不做复杂验证

**功能点**
- [ ] 按章节生成学习内容
- [ ] 内容包含：目标、讲解、示例、总结
- [ ] 使用 RAG 检索相关资料片段
- [ ] 支持重新生成单个章节

**验收标准**
- 生成的内容符合固定模板
- 内容与资料相关，不跑题
- 示例清晰易懂

**状态**：待开发

---

### 1.5 章节学习

**需求描述**
用户按章节顺序学习课程内容。

**功能点**
- [ ] 显示章节列表和学习进度
- [ ] 进入章节查看学习内容
- [ ] 标记章节完成状态
- [ ] 记录学习时长
- [ ] 支持跳转到任意章节
- [ ] 断点续学（记住上次学习位置）

**验收标准**
- 章节内容排版清晰易读
- 进度能正确保存和恢复
- 刷新页面后能回到上次位置
- 学习时长统计准确

**状态**：待开发

---

### 1.6 练习题（极简版）

**需求描述**
每章节配套简单练习题，帮助巩固知识。

> [!tip] MVP 简化
> 只做选择题，每章 2-3 道，不做统计

**功能点**
- [ ] 生成选择题（2-3 道/章节）
- [ ] 提交答案并判断对错
- [ ] 显示正确答案和解析
- [ ] 记录答题记录

**验收标准**
- 题目与章节内容相关
- 答案判断准确
- 解析清晰易懂

**状态**：待开发

---

### 1.7 学习进度管理（简化版）

**需求描述**
记录和展示学习进度，支持断点续学。

> [!tip] MVP 简化
> 只用文字显示进度，不做图表可视化

**功能点**
- [ ] 显示整体学习进度（百分比文字）
- [ ] 显示已完成章节列表
- [ ] 显示累计学习时长
- [ ] 显示最后学习时间
- [ ] 一键跳转到当前学习位置

**验收标准**
- 进度计算准确
- 数据实时更新
- 页面刷新后数据不丢失

**状态**：待开发

### 2.1 资料推荐（可选流程）

**需求描述**
根据学习目标推荐合适的学习资料。

**功能点**

- [ ] 给出推荐理由
- [ ] 提供"跳过推荐"选项
- [ ] 记录用户选择的资料

**验收标准**
- 推荐内容与学习目标相关
- 可以跳过直接上传资料

**状态**：待开发

---

AI 生成大纲后，用户可以手动调整。

**功能点**

- [ ] 删除章节
- [ ] 修改章节标题
- [ ] 调整章节顺序（拖拽）
- [ ] 拆分章节
- [ ] 合并章节

**验收标准**
- 编辑操作流畅

**状态**：待开发

**需求描述**
AI 生成的内容质量不好时，可以手动修改。

**功能点**
- [ ] 编辑章节内容（富文本编辑器）

- [ ] 修改练习题

**验收标准**

### 2.4 错题记录与复习

**需求描述**
记录做错的题目，方便后续复习。

**功能点**
- [ ] 自动收集错题
- [ ] 显示错题列表
- [ ] 查看错题解析
- [ ] 标注薄弱知识点
- [ ] 推荐复习章节
- [ ] 错题重做

**验收标准**
- 错题能正确记录
- 可以按章节筛选错题
- 重做后更新记录

### 2.5 学习报告

**需求描述**
阶段性生成学习报告，了解学习效果。

**功能点**
- [ ] 显示学习时长统计
- [ ] 显示章节完成情况
- [ ] 显示练习题正确率
- [ ] 标注薄弱知识点

**验收标准**
- 数据统计准确
- 可视化清晰
- 建议有参考价值

**状态**：待开发

### 2.6 多资料支持

**需求描述**

**功能点**

## MVP 功能优化说明

> [!success] MVP 简化原则
> 为了快速验证产品价值，MVP 版本做了以下简化：

### 移除的功能

| 原功能 | 移除原因 | 计划 |
|--------|---------|------|
| PDF 文件解析 | 技术复杂度高，容易失败 | P1 实现 |
| 大纲版本管理 | 功能过重，MVP 不需要 | P1 实现 |
| 填空题 | 题型太多，先验证选择题 | P1 实现 |
| 答题统计分析 | 统计功能非核心 | P1 实现 |
| 进度可视化图表 | 文字显示足够 | P1 实现 |
| 内容完整性验证 | 可以手动检查 | P1 实现 |
| 解析进度显示 | 文件小，解析快 | P1 实现 |

### 简化的功能

| 功能 | 原设计 | MVP 版本 |
|------|--------|---------|
| 文件上传 | PDF/TXT/MD，50MB | 只支持 TXT/MD，10MB |
| 练习题 | 选择题 3-5 道 + 填空题 2-3 道 | 只有选择题 2-3 道 |
| 进度显示 | 进度条/环形图 | 百分比文字 |
| 章节编辑 | 富文本编辑器 | 简单文本编辑 |
| 大纲管理 | 版本控制 + 回滚 | 只保存当前版本 |

### MVP 核心流程

```mermaid
graph LR
    A[学习目标问答] --> B[上传资料 TXT/MD]
    B --> C[生成大纲]
    C --> D[生成课程内容]
    D --> E[章节学习]
    E --> F[做练习题]
    F --> G[查看进度]
    G --> E
```

> [!tip] 预计开发时间
> - 基础框架：1 周
> - 核心功能：3-4 周
> - 测试优化：1 周
> - **总计：5-6 周**
